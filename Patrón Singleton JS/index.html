El patrón Singleton limita el número de instancias de un objeto en particular a solo uno. Esta única instancia se llama singleton.Los singletons son útiles en situaciones donde las acciones de todo el sistema deben coordinarse desde un solo lugar central. Un ejemplo es un grupo de conexiones de base de datos. El grupo administra la creación, destrucción y duración de todas las conexiones de bases de datos para toda la aplicación, asegurando que no se pierdan las conexiones. Los Singleton reducen la necesidad de variables globales, lo que es particularmente importante en JavaScript porque limita la contaminación del espacio de nombres y el riesgo asociado de colisiones de nombres.
